1""" 

2This modules implements the simulated annealing heuristic, which is a hill-climbing algorithm 

3which can probabilistically accept a worse solution than the current one. The probability of 

4accepting a value worse than the current one decreases with the number of iterations: this 

5introduces the notion of the system's "temperature" (hence the analogy of metal's annealing). 

6The temperature is a value that decreases overtime according to a cooling schedule which defines 

7the temperature decrease per iteration. As the temperature lower, the probability of moving 

8upward (i.e. accepting a worse solution) decreases until it reaches 0 and the system does not 

9move anymore. 

10 

11As with all hill-climbing algorithm, there is the necessity of defining a point "neighborhood" by 

12using some distance in order to select the next candidate for evaluation. 

13 

14Also, the simulated annealing heuristic comes with the possibility of restarting the system, 

15i.e. setting the temperature back to its maximum value. There are many ways of restarting the 

16system and we have selected two of them: random restarts, which restart the system following a 

17Bernouilli law (whose parameter can be tuned) and energy threshold restarts which heat the system up 

18once the system's energy is under a certain threshold.
